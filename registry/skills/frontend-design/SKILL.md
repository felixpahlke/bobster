# Frontend Design

Use this skill when building or revising user-facing web interfaces.

## Principles

- Build the actual working product screen first, not a marketing page unless requested.
- Match the interface density and tone to the domain.
- Prefer clear layouts, stable dimensions, and familiar controls over decorative novelty.
- Use visual assets when the product, place, person, or state needs to be inspectable.
- Verify responsive behavior and text fitting across mobile and desktop viewports.

## Checklist

- Primary workflows are reachable from the first screen.
- Buttons use icons for common actions when an icon is clearer than text.
- Text does not overlap or overflow its container.
- Fixed-format controls use stable dimensions.
- The palette has meaningful contrast and is not a single-hue wash.
